#### 2️⃣ 创建第一个智能体
| 配置项 | 填写建议（校园百事通场景） | 注意事项 |
|--------|---------------------------|----------|
| **智能体名称** | 智慧校园助手 / 校园百事通 | 名称简洁易记，体现功能定位 |
| **描述** | "为师生提供校园信息查询、办事指引、学习支持的AI助手" | 描述清晰有助于后续优化 |

| **创建方式** | 推荐「标准创建」，便于逐项配置 | AI创建适合有明确需求描述时 |

#### 3️⃣ 核心配置项解析
```
🔧 运行模式选择：
   ├─ 单Agent模式：适合校园问答、信息查询等简单场景（本次培训选用）
   └─ 多Agents模式：适合复杂任务拆解，如"选课咨询+流程办理+结果反馈"

🧠 大模型选择：
   ├─ 云雀语言模型（Coze默认）：响应快，适合中文场景
   ├─ Moonshot（Kimi）：长文本理解强，适合政策解读类问题
   └─ 建议：初期选用云雀32K版本，平衡成本与效果

⚙️ 关键参数设置：
   • Temperature（随机性）：0.3-0.5（校园问答需准确，不宜过高）
   • Max Tokens（回复长度）：2048（平衡信息量与加载速度）
   • 上下文轮数：5-8轮（保证多轮对话连贯性）
```

## 📅 9:30-10:45｜提示词工程核心技巧

### 🔹 学习目标
- 掌握提示词（Prompt）设计的三大原则：清晰、具体、结构化
- 学会运用角色设定、格式约束、示例引导等进阶技巧
- 理解系统提示词与用户提示词的区别与应用场景

### 🔹 提示词设计黄金法则

#### 🎯 原则一：清晰明确（Clarity）
```
❌ 模糊表述："你是一个校园助手，回答学生问题"
✅ 清晰表述："你是XX大学官方智慧校园助手，专门解答师生关于教务、后勤、图书馆、校园卡等校内事务的咨询"
```

#### 🎯 原则二：具体可执行（Specificity）
```
❌ 笼统要求："回答要友好"
✅ 具体指令：
   • 语气：亲切专业，使用"同学""老师"等校园称呼
   • 结构：先结论后解释，复杂问题分步骤说明
   • 边界：涉及个人隐私/敏感信息时，引导至官方渠道
```

#### 🎯 原则三：结构化表达（Structured）
```markdown
# Role: 智慧校园助手
* author: 校园信息化中心
* version: 1.0
* language: 中文

## Profile
你是XX大学官方AI助手，熟悉校园规章制度、办事流程、设施分布等信息，致力于为师生提供准确、高效、有温度的服务。

## Goals
1. 准确解答校园生活、学习、行政类问题
2. 引导用户通过正确渠道办理复杂业务
3. 对不确定信息明确说明，不编造内容

## Skills
### 技能1：校园信息查询
- 支持查询：课表、考试安排、图书馆开放时间、食堂菜单等
- 数据来源：优先调用知识库，无结果时说明"暂未收录"

### 技能2：办事流程指引
- 分步骤说明：学生证补办、宿舍申请、请假流程等
- 附带：所需材料、办理地点、工作时间、咨询电话

## Constraints
⚠️ 必须遵守：
• 不涉及政治、宗教等敏感话题
• 不替代官方通知，重要信息建议用户核实官网
• 保护用户隐私，不存储个人身份信息

## Output Format
• 简短问题：直接回答+1句补充建议
• 复杂问题：分点说明+关键信息加粗+相关链接
• 无法回答：诚恳说明+推荐咨询渠道
```

### 🔹 进阶技巧实战

#### ✨ 技巧1：角色设定增强代入感
```
🎭 基础版："你是一个助手"
🎭 进阶版：
"你是陪伴XX大学师生成长的智能伙伴'小园'，性格阳光耐心，像学长学姐一样既专业又亲切。
当同学焦虑时，先共情再给方案；当老师咨询时，语言简洁专业。"
```

#### ✨ 技巧2：格式约束提升可用性
```
📋 要求结构化输出：
"请按以下格式回答选课问题：
【核心结论】一句话总结
【操作步骤】1️⃣ 2️⃣ 3️⃣ 分步说明
【注意事项】⚠️ 关键提醒
【延伸阅读】🔗 相关链接/政策文件"
```

#### ✨ 技巧3：示例引导（Few-shot Learning）
```
💡 在提示词末尾添加示例：
## Examples
用户：图书馆周末开放吗？
助手：【开放信息】XX校区图书馆周末正常开放，时间8:00-22:00 📚
      【温馨提示】期末期间座位紧张，建议通过"图书馆公众号"提前预约~

用户：学生证丢了怎么办？
助手：【办理流程】学生证补办分3步：
      1️⃣ 登录教务系统→"证件管理"→在线挂失
      2️⃣ 携带身份证+1寸照片，到行政楼103提交申请
      3️⃣ 5个工作日后凭回执领取新证
      ⚠️ 补办工本费20元，仅支持校园卡支付
```

> 📚 **学习资源**：Coze平台内置"提示词优化"功能，可输入初稿后点击"优化"获取AI建议；平台案例库提供多场景Prompt模板参考。

## 📅 10:45-12:00｜实战演练：智能体人设优化

### 🔹 学习目标
- 为"智慧校园助手"编写完整人设提示词
- 通过A/B测试对比不同提示词效果
- 掌握迭代优化的方法论与评估标准

### 🔹 实战任务流程

#### 📝 阶段1：初版提示词编写（20分钟）
```
🎯 任务：基于上午所学，为校园百事通编写系统提示词
✅ 必含要素：
   • 角色定位（身份+服务对象+核心价值）
   • 能力边界（能做什么/不能做什么）
   • 交互风格（语气+格式+响应逻辑）
   • 校园特色（融入学校文化/常用术语）

📋 参考框架：
# Role: [你的智能体名称]
## 我是谁 → ## 我能帮什么 → ## 我怎么回答 → ## 我的底线
```

#### 🔄 阶段2：A/B测试与效果对比（25分钟）
```
🔬 测试方法：
1️⃣ 保存初版提示词为"V1.0"
2️⃣ 修改1-2个变量（如：语气更活泼/增加示例/调整输出格式）
3️⃣ 保存为"V1.1"，在相同测试问题下对比输出

🧪 推荐测试问题库：
• 基础查询："图书馆几点关门？"
• 流程咨询："如何申请自习室？"
• 模糊问题："最近有什么活动？"（测试澄清能力）
• 边界问题："帮我写期末论文"（测试规则遵守）

📊 评估维度：
✓ 准确性：信息是否正确、来源是否可靠
✓ 完整性：是否覆盖用户核心需求
✓ 友好度：语气是否符合校园场景
✓ 引导性：复杂问题是否给出清晰路径
```

#### 🚀 阶段3：迭代优化与最佳实践总结（30分钟）
```
🔧 优化策略：
• 问题：回答太冗长 → 方案：在Constraints中添加"简短问题回答不超过3句话"
• 问题：专业术语过多 → 方案：在Profile中加入"用大一新生能理解的语言解释"
• 问题：无法处理多意图 → 方案：在Workflow中增加"先拆分问题，再逐一回答"

✨ 校园场景特调技巧：
1️⃣ 融入校园梗：适当使用"早八人""干饭"等学生常用语，拉近距离
2️⃣ 时效性提示：涉及校历/考试/放假等问题，自动标注"信息截至2026年X月"
3️⃣ 情感支持：对"挂科焦虑""选课困难"等问题，先共情再给方案
4️⃣ 多模态预留：在Output Format中注明"如需地图/表格，可调用插件生成"

📌 交付成果：
□ 优化后的完整提示词（V2.0版）
□ 3组对比测试截图+效果分析
□ 个人优化心得（100字内）
```

### 🔹 常见误区与避坑指南
```
⚠️ 误区1：提示词越长越好
→ 正解：精炼结构化 > 冗长堆砌，核心规则前置

⚠️ 误区2：一次写出完美提示词
→ 正解：小步快跑，每轮只优化1-2个点，记录变化效果

⚠️ 误区3：忽略知识库联动
→ 正解：在提示词中明确"优先查询知识库，无结果再通用回答"

⚠️ 误区4：过度拟人化导致越界
→ 正解：亲切但不承诺能力边界，敏感问题引导至人工渠道
```
