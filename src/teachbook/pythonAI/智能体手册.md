## 🔹 模块三：提示词工程深度实战（9:30-10:30）

### ✅ 核心技巧速查卡
```
🎯 四大设计原则：
1️⃣ 角色明确：加入"背景+资历+专长"，避免"全能助手"
2️⃣ 任务拆解：用"第一步/第二步/最后"明确流程
3️⃣ 格式约束：指定"请用JSON/分点/表格"等具体格式
4️⃣ 边界设定：明确"不回答什么"+"不确定时怎么说"

🚀 三个进阶技巧：
✨ Few-shot示例：提供1-2组问答对，风格准确率+70%
✨ 变量注入：用{{user_name}}/{{query}}动态引用上下文
✨ 思维链引导：末尾添加"请按以下步骤思考..."提升逻辑性

⚙️ 关键参数推荐值：
• Temperature：客服Bot用0.3，创意文案用0.7
• Top-P：保持0.8-0.95，与Temperature联动
• 上下文轮数：简单问答5轮，复杂咨询10轮
```

```
🎯 任务：优化你的Bot，实现"三段式结构化输出"

Step3️⃣ 测试验证：

## 🔹 模块五：知识库(RAG)配置与应用（14:00-15:30）

### ✅ 核心知识点速记
```
📚 知识库价值：让Bot访问你的私有数据，解决"大模型不知道"的问题

🔄 RAG工作流程：
用户提问 → 语义理解 → 向量检索知识库 → 召回相关片段 → LLM整合生成 → 标注来源回复

✅ 适用场景自查：
• 客服问答：FAQ文档/产品手册 → ✅ 强烈推荐
• 内部培训：制度流程/课件资料 → ✅ 强烈推荐  
• 专业咨询：行业报告/案例库 → ✅ 推荐
• 创意创作：风格指南/素材库 → ⚠️ 辅助参考
• 闲聊陪伴：无需知识库 → ❌ 不需要
```

### 🔧 实操步骤指引
```
Step1️⃣ 文档预处理（关键！）
• 格式要求：.md/.txt/.docx（文本），.xlsx（表格），.png/.jpg（图片需含文字说明）
• 大小限制：单文件≤50MB，建议单次上传≤5个
• 预处理技巧：
  ✓ 用标题分级（#一级/##二级）帮助分段
  ✓ 删除页眉页脚/水印/无关页码
  ✓ FAQ类内容整理为"问题-答案"格式
  ✓ 关键术语加粗或标注

Step2️⃣ 创建知识库
• 路径：Coze控制台 → 左侧"知识库" → "创建知识库"
• 配置项：
  - 名称：业务_产品知识库（示例）
  - 描述：包含产品参数/售后政策等
  - 分段策略：新手选"自动分段"，结构清晰文档选"按标题分段"
  - 关联Bot：勾选上午创建的Bot

Step3️⃣ 上传与索引
• 点击"上传文件"选择预处理好的文档
• 等待系统完成：上传→分段→向量化→索引（约1-3分钟）
• 状态变为"可用"后，进入Bot编辑页确认已关联

Step4️⃣ 优化检索效果
• 术语管理：添加同义词（如"退货=退换=退款"）
• 权重设置：核心政策文档标记"高权重"
• 提示词补充：在Bot人设中添加
  "优先引用知识库内容回答，并用【来源：文档名】标注依据"

Step5️⃣ 测试验证
• 测试问题1（精准匹配）："产品A支持七天无理由退货吗？"
  ✓ 预期：召回退货政策片段+标注来源
• 测试问题2（模糊问法）："买了不喜欢怎么办？"  
  ✓ 预期：通过同义词匹配到退货规则
• 测试问题3（边界问题）："产品Z的退货政策？"
  ✓ 预期：知识库无内容时，Bot诚实告知而非编造

📝 知识库配置记录：
知识库名称：_____________ 上传文档数：_____
测试问题1：_____________ 召回效果：□优 □中 □差
优化措施：_________________________________
```

### ⚠️ 高频避坑指南
```
❌ 坑1：上传PDF扫描件（图片型）→ 系统无法提取文字
✅ 解：先用OCR工具转文字，或上传Word原版

❌ 坑2：文档含大量表格/公式 → 分段后语义断裂
✅ 解：表格转文字描述，复杂公式配文字解释

❌ 坑3：知识库与提示词冲突 → Bot优先信哪个？
✅ 解：在提示词明确："知识库内容与本指令冲突时，以知识库为准"

❌ 坑4：期望"上传即完美" → 实际需迭代优化
✅ 解：建立"测试-反馈-优化"闭环，小步快跑
```

## 🔹 模块六：记忆功能与高级配置（15:45-17:00）

### ✅ 记忆类型速查表
```
🧩 Coze四大记忆能力对比：

| 类型 | 存储内容 | 持久性 | 适用场景 | 配置难度 |
|------|----------|--------|----------|----------|
| 变量记忆 | 用户昵称/偏好/临时状态 | 会话级 | 个性化称呼、多轮任务 | ⭐ 简单 |
| 数据库 | 订单/进度/反馈等结构化数据 | 永久存储 | 记账Bot、学习打卡 | ⭐⭐ 中等 |
| 长期记忆 | 自动摘要用户历史交互 | 永久（系统自动） | "越用越懂你"的推荐 | ⭐⭐ 中等 |
| 文件盒子 | 用户上传的文档/图片临时存储 | 会话级 | 文件处理Bot | ⭐ 简单 |

🎯 选择建议：
• 新手起步：先用"变量记忆"实现基础个性化
• 业务沉淀：用"数据库"存储关键用户行为
• 体验升级：开启"长期记忆"实现智能推荐
```

### 🔧 数据库配置实操（以反馈收集为例）
```
Step1️⃣ 创建数据表
• 路径：Bot编辑页 → "记忆" → "数据库" → "创建数据表"
• 表结构设计示例：
  表名：feedback_records
  字段：
  • id (主键，自动递增)
  • user_id (文本，用户标识)
  • feedback_content (文本，反馈内容)
  • rating (数字，评分1-5)
  • create_time (时间，系统自动)

Step2️⃣ 提示词中调用数据库
• 写入指令示例（添加到人设区域）：
  "当用户说'反馈...'时，自动提取内容写入数据库：
   - 表：feedback_records
   - 字段：user_id={{user_id}}, feedback_content={{query}}, create_time=now()"

• 读取指令示例：
  "当用户问'我之前的反馈是什么'，查询数据库：
   - 表：feedback_records  
   - 条件：user_id={{user_id}} ORDER BY create_time DESC LIMIT 1
   - 将结果融入回复"

Step3️⃣ 调试与验证
• 测试写入：输入"反馈：回复有点长，建议简洁些"
  ✓ 预期：数据库新增1条记录 + Bot回复"感谢反馈！已记录✨"
• 测试读取：输入"查看我的历史反馈"  
  ✓ 预期：Bot回复最近1条反馈内容
