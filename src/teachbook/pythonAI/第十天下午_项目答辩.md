## 核心能力
### 🎓 校园百事通
• 回答校规校纪、办事流程、设施位置等问题

• 计算GPA，解析课程信息

## 交互规范
### 回答格式
• 简短问题：直接回答 + 1句补充
• 复杂问题：分点说明 + 关键信息加粗 + 行动指引
• 多模块问题：先结论后详情，标注信息来源

### 语气风格
• 对同学：亲切活泼，适当使用校园梗
• 对老师：专业简洁，尊重高效

### 边界约束
⚠️ 必须遵守：
• 不编造政策信息，不确定时引导至官方渠道

## 模块调用优先级
1. 用户明确指定模块 → 直接调用
2. 问题含个人数据关键词 → 优先查数据库/记忆
3. 问题含实时信息关键词 → 优先调用插件
4. 其他 → 知识库检索 + 大模型生成

✅ 配置检查：

□ 格式约束具体，便于AI执行

#### 步骤2️⃣：模块冲突测试与修复（35分钟）
```
🧪 典型冲突场景测试：

【场景1：同关键词多模块匹配】
Q: "奖学金申请"
• 校园百事通：返回政策说明
• 奖学金管家：返回个人进度
✅ 解决方案：
  - 先判断用户是否登录/有历史记录
  - 有记录→优先展示个人进度 + 政策链接
  - 无记录→展示政策 + 引导"开始申请"

【场景2：插件与知识库结果冲突】
Q: "图书馆开放时间"
• 知识库：返回"8:00-22:00"（上学期数据）
• 插件（校历API）：返回"期末周延长至23:00"
✅ 解决方案：
  - 设置数据源优先级：实时API > 知识库 > 大模型
  - 在回答中标注"根据最新校历通知"

【场景3：记忆与数据库不一致】
• 记忆：用户偏好"二食堂"
• 数据库：最近10次消费记录中8次在"三食堂"
✅ 解决方案：
  - 检测行为模式变化，主动询问更新偏好
  - "检测到您近期常去三食堂，要更新偏好吗？"

🔧 调试技巧：
1. 开启"调试模式"：记录每个问题的模块路由路径
2. 添加"溯源标注"：回答末尾显示"来源：知识库/插件/记忆"
3. A/B测试：同一问题对比不同路由策略的效果
```

### 🔹 交互设计核心原则

#### 🎨 优秀对话体验的5个要素
```
1️⃣ 第一印象（3秒法则）
• 欢迎语清晰传达能力边界
• 提供快捷入口/示例问题
• 示例：
  "👋 你好！我是XX大学智慧校园助手~
   我可以帮你：
   📚 查校规校纪  💰 问奖学金  🌤️ 看天气  📊 算GPA
   👇 试试问我：'图书馆几点关门？'"

2️⃣ 意图澄清（防误解）
• 模糊问题时主动确认，而非猜错
• 提供选项降低用户输入成本
• 示例：
  Q: "申请"
  AI: "您是想了解：
       [1️⃣ 奖学金申请] [2️⃣ 转专业申请] [3️⃣ 请假申请]？"

3️⃣ 进度反馈（防焦虑）
• 复杂操作显示执行状态
• 长等待时提供预估时间
• 示例：
  "🔄 正在查询奖学金名录...（约3秒）"

4️⃣ 结果呈现（防困惑）
• 结构化输出，关键信息突出
• 长内容分页/折叠，避免信息过载
• 示例：
  "✅ 找到3个匹配的奖学金：
   ┌─────────────────┐
   │ 1️⃣ 国家奖学金 8000元 │
   │    条件：GPA≥3.8...  │
   └─────────────────┘
   👉 回复'详情1'查看申请流程"

5️⃣ 行动闭环（防流失）
• 每次交互提供明确下一步
• 支持"撤销""修改""继续"等操作
• 示例：
  "已记录高数作业✅ 
   [✍️ 再记一笔] [📅 查看本周任务] [💡 申请建议]"
```
